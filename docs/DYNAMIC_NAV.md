# Dynamic Navigation System - Implementation Guide

## Overview

The documentation site now uses a **dynamic navigation system** that automatically generates sidebar menus based on your folder structure. No more manual editing of navigation code!

## How It Works

### 1. Navigation Generator Script

**File**: `scripts/generate_nav.rb`

This Ruby script:

- Scans all folders in the `/docs` directory
- Finds all markdown files in each folder
- Extracts page titles from YAML front matter
- Generates `_data/navigation.yml` with the complete structure

### 2. Data File

**File**: `_data/navigation.yml` (auto-generated)

Contains two main sections:

- **main**: Top-level sections (displayed on home page sidebar)
- **sections**: Navigation items for each section

Example structure:

```yaml
main:
  - title: Installation
    url: /installation/
    folder: installation

sections:
  installation:
    - title: Installation Overview
      url: /installation/
    - title: Getting Started
      url: /installation/getting-started/
```

### 3. Dynamic Layout

**File**: `_layouts/default.html`

The layout now uses Liquid logic to:

- Detect the current page's section
- Display section-specific navigation on section pages
- Display all sections on the home page
- Automatically highlight the active page

## Workflow

### Adding a New Page

1. Create the markdown file:

   ```bash
   echo '---
   layout: default
   title: My New Page
   ---

   Page content here' > docs/pipeline/my-new-page.md
   ```

2. Regenerate navigation:

   ```bash
   ruby docs/scripts/generate_nav.rb
   ```

3. The page automatically appears in the sidebar!

### Adding a New Section

1. Create a folder:

   ```bash
   mkdir docs/my-section
   echo '---
   layout: default
   title: My Section
   ---' > docs/my-section/index.md
   ```

2. Run the generator:

   ```bash
   ruby docs/scripts/generate_nav.rb
   ```

3. Your section appears in the home page sidebar!

### Development Workflow

Use the helper script for seamless development:

```bash
bash docs/scripts/dev.sh
```

This automatically:

1. Generates fresh navigation
2. Installs dependencies if needed
3. Starts Jekyll with incremental builds

## Important Notes

### Title in Front Matter

The generator reads page titles from markdown front matter:

```yaml
---
layout: default
title: This Title Appears in Sidebar
---
```

If no title is found, it uses the filename (converted to title case).

### Regeneration Required

The navigation file is NOT automatically regenerated by Jekyll. You must run the script:

```bash
ruby docs/scripts/generate_nav.rb
```

This is intentional to:

- Keep builds fast
- Avoid scanning on every serve
- Give you control over when navigation updates

### Git Workflow

The `.gitignore` includes `_data/navigation.yml`. This means:

- The script is committed to git
- The generated file is NOT committed
- Other developers run the script after pulling changes

If you prefer to commit the generated file, simply remove it from `.gitignore`.

## Customization

### Changing Navigation Order

The `generate_nav.rb` script sorts items alphabetically. To customize ordering, you can:

1. Edit the generated `_data/navigation.yml` manually (but it will be overwritten next time you run the script)
2. Modify `generate_nav.rb` to respect a config file for custom ordering
3. Rename your folders/files with numeric prefixes (e.g., `01-installation`, `02-pipeline`)

### Custom Front Matter Processing

To add more metadata to navigation items, edit `generate_nav.rb` to extract additional front matter fields like:

- `description` - Short description for tooltips
- `order` - Custom sort order
- `hidden` - Hide from navigation

## Troubleshooting

### Navigation not updating

Make sure you ran:

```bash
ruby docs/scripts/generate_nav.rb
```

### New page doesn't appear

1. Check the file is in the correct section folder
2. Ensure it has valid front matter with `title`
3. Run the generator script
4. Restart Jekyll

### Duplicate pages appearing

Clear Jekyll cache:

```bash
rm -rf docs/_site docs/.jekyll-cache
ruby docs/scripts/generate_nav.rb
bundle exec jekyll serve
```
